## 当前判断

- 路由级懒加载已经生效，页面包整体较小。
- 最大问题是主包 `index-*.js` 超过 500 kB，主要可能来自 AntD 运行时、共享依赖和全局引入。
- Chat 页面包在可接受范围内，但仍是后续分析重点，因为包含 Markdown 渲染和聊天相关组件。

## 下一轮优化方向

1. 优先分析 `index-*.js` 主包来源，确认 AntD 与图标是否存在可拆分空间。
2. 检查 `@ant-design/icons` 的导入是否保持按需引用，避免从 barrel 引入过多图标。
3. 保持当前 `React.lazy` 路由级拆分，不回退为同步页面导入。
4. 如需精确定位，再引入 bundle analyzer，不在本轮直接做高风险拆包。
5. 若部署后首屏加载慢，再考虑对非首屏依赖做 hover/focus preload 或延迟加载。

## 后续预算建议

暂定个人作品集首版预算：

- 主包 gzip 目标：小于 180 kB，当前约 180.28 kB，处于边界。
- 单页面包 gzip 目标：小于 50 kB，当前均满足。
- CSS gzip 目标：单文件小于 5 kB，当前均满足。

下一轮如果继续优化，应先引入分析工具定位来源，而不是盲目拆代码。
